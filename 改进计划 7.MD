## 问题背景

### 核心问题描述

当前应用中存在多个 View 重复创建相同 Service 实例的问题，导致：

1. **性能浪费** - 每次初始化都会执行异步数据加载
2. **数据不一致** - 多个实例可能持有不同的数据状态
3. **资源浪费** - 重复的 CoreData 上下文和网络请求

### 当前问题代码

```swift
// CommunityView.swift - Line 11-12
@StateObject private var achievementService = AchievementService()  // ❌ 新实例
@StateObject private var rankingService = PersonalRankingService()   // ❌ 新实例

// ProfileView.swift - Line 11
@StateObject private var viewModel = ProfileViewModel()  // ✅ 无问题

// DashboardView.swift - Line 13
@StateObject private var achievementService = AchievementService()  // 主实例
```

### 依赖关系图

```
ContentView (TabView)
├── DashboardView
│   └── @StateObject AchievementService ← 主实例 ✓
├── CommunityView
│   └── @StateObject AchievementService ← 重复实例 ❌
│   └── @StateObject PersonalRankingService ← 重复实例 ❌
└── SettingsView
    └── (无 Service)
```

## 优化原则

### 1. 需求描述、生成规则和检验标准

| 需求项 | 描述 | 检验标准 |
|--------|------|----------|
| **DI-001** | 所有共享 Service 通过 ContentView 统一管理 | 代码审查 |
| **DI-002** | Tab 子页面通过 EnvironmentObject 接收 Service | 编译通过 |
| **DI-003** | 旧实例创建方式标记 @available(deprecated) | 编译器警告 |
| **DI-004** | 优化后首屏加载时间减少 ≥30% | 性能测试 |

### 2. 功能模块原则（单一职责）

```
优化后文件结构:
├── ContentView.swift                    # 服务容器（单一职责：管理共享服务）
├── ContentView+Services.swift            # 服务初始化扩展
├── CommunityView.swift                  # 移除 @StateObject，使用 @EnvironmentObject
├── DashboardView.swift                  # 提供服务给子视图
└── ProfileView.swift                    # 无需修改（无共享服务）

模块职责划分:
- ContentView: 负责创建和持有共享 Service 实例
- 子View: 通过 @EnvironmentObject 消费服务，不负责创建
- Service: 保持现有职责，不做修改
```

### 3. 代码复用原则

| 场景 | 处理方式 | 依据 |
|------|----------|------|
| AchievementService | 复用现有实例 | 已存在三次以上使用 |
| PersonalRankingService | 复用现有实例 | 业务需要 |
| ProfileViewModel | 保持独立 | 职责不同，仅在此处使用 |

### 4. 废弃代码清理规则

```
清理流程:
1. 标记废弃 → 使用 @available(deprecated) 包装旧 init
2. 验证无影响 → 运行测试确认功能正常
3. 删除代码 → 移除 @StateObject 创建的旧实例
4. 提交信息 → "refactor: remove deprecated service initialization in CommunityView"
```

### 5. 代码风格一致性

- 保持 SwiftUI 现有命名规范
- 使用与 LogActivityView 相同的依赖注入模式
- 遵循 AppColors、Typography 等现有设计系统

### 6. 继承关系

```
继承链（而非硬编码）:
ContentView
    ↓ provides @StateObject
DashboardView
    ↓ already has achievementService
LogActivityView (已修复)
    ↓ needs service
CommunityView (待优化)
    ↓ uses
AchievementService (已存在)
    ↓ uses
CoreDataAchievementRepository (已存在)
```

## 技术方案

### 方案对比

| 方案 | 优点 | 缺点 | 推荐 |
|------|------|------|------|
| **A: EnvironmentObject** | Apple 官方推荐，无需修改子 View | 需要 ContentView 作为容器 | ⭐ 推荐 |
| **B: 自定义 DI 容器** | 完全控制 | 需要大量重构 | ❌ 不推荐 |
| **C: Singleton** | 简单直接 | 隐藏依赖，难以测试 | ❌ 不推荐 |

### 方案 A 实施步骤

```
Step 1: ContentView 添加 @StateObject 服务
Step 2: 子 View 使用 @EnvironmentObject
Step 3: 标记 CommunityView 旧 init 为废弃
Step 4: 验证编译和运行
Step 5: 清理废弃代码
```

## 功能模块设计

### 模块 1: ServiceContainer（服务容器）

**职责**: 在应用根节点创建和持有共享服务实例

**文件**: ContentView+Services.swift（扩展）

```swift
// 扩展 ContentView 添加服务管理
extension ContentView {
    static let sharedServices = ServiceContainer()
}

struct ServiceContainer {
    let achievementService: AchievementService
    let rankingService: PersonalRankingService
    
    init() {
        self.achievementService = AchievementService()
        self.rankingService = PersonalRankingService()
    }
}
```

**检验标准**:
- [ ] ServiceContainer 在 ContentView 初始化时创建
- [ ] 所有共享服务只创建一次

### 模块 2: Environment Injection（环境注入）

**职责**: 将服务注入到 SwiftUI 环境

**文件**: ContentView.swift（修改）

```swift
struct ContentView: View {
    @StateObject private var achievementService = AchievementService()
    @StateObject private var rankingService = PersonalRankingService()
    
    var body: some View {
        TabView {
            DashboardView(achievementService: achievementService)
                .environmentObject(achievementService)
                .tabItem { ... }
            
            CommunityView(
                achievementService: achievementService,
                rankingService: rankingService
            )
                .tabItem { ... }
            
            SettingsView()
                .tabItem { ... }
        }
    }
}
```

**检验标准**:
- [ ] TabView 中所有 View 都能访问 @EnvironmentObject
- [ ] 编译无错误

### 模块 3: CommunityView 依赖注入

**职责**: 使用注入的服务而非创建新实例

**文件**: CommunityView.swift（修改）

```swift
// 修改前 ❌
@StateObject private var achievementService = AchievementService()
@StateObject private var rankingService = PersonalRankingService()

// 修改后 ✅
let achievementService: AchievementService
let rankingService: PersonalRankingService

// 废弃 init（兼容旧代码）
@available(*, deprecated, message: "Use init with injected services instead")
init() {
    self.achievementService = AchievementService()
    self.rankingService = PersonalRankingService()
}

// 新 init
init(
    achievementService: AchievementService,
    rankingService: PersonalRankingService
) {
    self.achievementService = achievementService
    self.rankingService = rankingService
}
```

**检验标准**:
- [ ] CommunityView 接收外部注入的服务
- [ ] @StateObject 移除，服务通过 init 注入

## 代码实现

### 1. ContentView.swift 修改

```swift
//
//  ContentView.swift
//  suslife
//
//  Main Entry Point - Tab-based Navigation
//  Modified: 添加共享服务实例
//

import SwiftUI

struct ContentView: View {
    @StateObject private var achievementService = AchievementService()
    @StateObject private var rankingService = PersonalRankingService()
    
    var body: some View {
        TabView {
            DashboardView(achievementService: achievementService)
                .tabItem {
                    Label("Home", systemImage: "house.fill")
                }
            
            CommunityView(
                achievementService: achievementService,
                rankingService: rankingService
            )
                .tabItem {
                    Label("My Progress", systemImage: "chart.line.uptrend.xyaxis")
                }
            
            SettingsView()
                .tabItem {
                    Label("Settings", systemImage: "gearshape.fill")
                }
        }
        .accentColor(AppColors.primary)
    }
}
```

### 2. DashboardView.swift 修改（添加 init）

```swift
// 在现有 struct 中添加 init（保持向后兼容）
let achievementService: AchievementService

init(achievementService: AchievementService) {
    self.achievementService = achievementService
}

// 保留原有 init 兼容预览
@available(*, deprecated)
init() {
    self.achievementService = AchievementService()
}
```

### 3. CommunityView.swift 修改

```swift
//
//  CommunityView.swift
//  suslife
//
//  Community View - Personal progress tracking
//  Modified: 使用依赖注入的服务实例
//

import SwiftUI

struct CommunityView: View {
    // 修改：移除 @StateObject，使用 let 接收注入的服务
    let achievementService: AchievementService
    let rankingService: PersonalRankingService
    
    // 直接数据来自 repository
    @State private var totalCO2Saved: Double = 0
    @State private var totalActivities: Int = 0
    @State private var currentStreak: Int = 0
    @State private var isLoading = false
    @State private var errorMessage: String = ""
    @State private var showErrorAlert = false
    @State private var showRetryButton = false
    
    private let repository = CoreDataActivityRepository()
    
    // 新 init：依赖注入
    init(
        achievementService: AchievementService,
        rankingService: PersonalRankingService
    ) {
        self.achievementService = achievementService
        self.rankingService = rankingService
    }
    
    // 废弃 init：兼容旧代码和预览
    @available(*, deprecated, message: "Use init with injected services")
    init() {
        self.achievementService = AchievementService()
        self.rankingService = PersonalRankingService()
    }
    
    var body: some View {
        ScrollView {
            // ... 现有代码保持不变
        }
        .task {
            await loadData()
        }
    }
}
```
